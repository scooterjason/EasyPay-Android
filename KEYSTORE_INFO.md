## Usage

### Building Release APK

```bash
./gradlew assembleRelease
```

The build system automatically uses the keystore configured in `app/build.gradle`:

```gradle
signingConfigs {
    release {
        storeFile file("easypay-release-key.jks")
        storePassword "EasyPay@2024"
        keyAlias "easypay"
        keyPassword "EasyPay@2024"
    }
}
```

### Verifying Keystore

```bash
keytool -list -v -keystore app/easypay-release-key.jks -storepass EasyPay@2024
```

### Exporting Certificate

```bash
keytool -export -alias easypay -keystore app/easypay-release-key.jks \
  -storepass EasyPay@2024 -file easypay-cert.cer
```

## Security Best Practices

1. **Backup**: Keep a secure backup of this keystore file
2. **Access Control**: Restrict access to authorized developers only
3. **Version Control**: Add `*.jks` to `.gitignore` - never commit keystores
4. **Environment Variables**: Store passwords in secure environment variables for CI/CD
5. **Rotation**: Consider rotating the keystore every 2-3 years
6. **Monitoring**: Track who has access to the keystore

## Troubleshooting

### "Keystore was tampered with, or password was incorrect"

- Verify the keystore file is not corrupted
- Check the password is correct
- Ensure the file path is correct

### "Key was not generated by this keystore"

- Verify the key alias is correct (`easypay`)
- Check the keystore file contains the key

### "Certificate has expired"

- The keystore is valid until June 24, 2053
- If expired, generate a new keystore with updated date

## Generating a New Keystore (If Needed)

If you need to generate a new keystore with different credentials:

```bash
keytool -genkey -v -keystore app/easypay-release-key.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias easypay \
  -storepass YOUR_NEW_PASSWORD \
  -keypass YOUR_NEW_PASSWORD \
  -dname "CN=EasyPay,O=EasyPay Inc,L=San Francisco,ST=California,C=US"
```

⚠️ **Warning**: Generating a new keystore will create a different signing certificate. Apps signed with the old keystore cannot be updated with the new one. Only do this if absolutely necessary.
